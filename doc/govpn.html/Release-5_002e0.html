Release 5.0

  • New optional encryptionless mode of operation. Technically no encryption functions are applied for outgoing packets, so you can not be forced to reveal your encryption keys or sued for encryption usage.
  • MTUs are configured on per-user basis.
  • Simplified payload padding scheme, saving one byte of data.
  • Ability to specify TAP interface name explicitly without any up-scripts for convenience.
  • govpn-verifier utility also can use EGD.
